Documentation

My Flix is a web application in which users can search for and save information about movies (different movies, directors and genres). Users will be able to sign up, update their personal information and create a list of their favorite movies.


Web server documentation - REST API Movie and User requests
REQUEST - Business Logic URL HTTP Method Request Data Format Response Data Body Format
Return a list of All movies to the user /movies GET None A JSON object holding data about all movies
Return data (description, genre, director, Image url, whether it's featured or not) about a single movie by title to the user… /movies/[Title] GET None A JSON object holding data about a single movie, containing an ID, title, and some other properties. Example:

{
_id: "5e9da6faaa98398565197490",
title: "Jurassic Park",
description: "The film is set on the fictional island .... created a wildlife park of de-extinct dinosaurs.",
genre: "Adventure",
director: "Steven Spielberg",
imagePath: "",
Featured: true
}

Return data about All genres (description) by name movies/Genre/[Name] GET None A JSON object holding data about all genres
Return data about a director (bio, birth year, death year) by name /movies/directors/[Name] GET None A JSON object holding data about a single director, containing name, birth year, death year, movies. Example:

{
name: "John Landis",
bio: " John Landis is an American film director, screenwriter, actor,…, music videos with singer Michael Jackson. " Birth "1950",
death: "",
}

Allow new users to register /users POST A JSON object holding data about the new user to register/add, structured like:

{
Username : "Ruby Took",
Password : "peiQu0hi4",
Email : "RubyTook@teleworm.us",
Birthday : "1988-03-31"
}

A JSON object holding data about the new user to register plus ID, structured like:

{
_id : 5e9c54713060a1071e806910"",
Username : "Ruby Took",
Password : "peiQu0hi4",
Email : "RubyTook@teleworm.us",
Birthday : "1988-03-31",
}
Allow users to update their user info (username, password, email, date of birth) /users/[Username] PUT A JSON object holding data about the current user to update, structured like:

{
"Username" : "Romy Took",
"Password" : "peiQu0hi4",
"Email" : "RubyTook@teleworm.us",
"Birthday" : "1988-03-31",
"Favorites" : ["movie_id"] }

A text message indicating whether the user's account has succesfully changed.
Allow users to add a movie to their list of favorites /users/[Username]/ Favorites/[movie_id] POST A JSON object holding data about the movie to add to the user's list of favorite movies, structured like:

{
"_id": "5e9c54713060a1071e806910,
"username": "Ruby Took",
"password": "peiQu0hi4",
"email": "RubyTook@teleworm.us",
"birthday": "1988-03-31",
"favorites": [ ObjectId("5e9c365e3060a1071e806907")]
}

A text message indicating whether the user's favorite movie was succesfully added to their list.
Allow users to remove a movie to their list of favorites /users/[Username]/Favorites/[movie_id] DELETE None A text message indicating whether the movie was succesfully removed from the users' favorite-list.
Allow existing users to deregister (remove a user by Username or id) /users/[Username]/ DELETE None A text message indicating whether the user's account was succesfully removed.
